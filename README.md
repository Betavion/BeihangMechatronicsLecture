### 每个案例程序目录说明

每个实验工程目录内部，通常可以分成“自动生成部分”和“自定义代码部分”两类：

- **自动生成部分**
  - `*.ioc`：STM32CubeMX 工程配置文件
  - `.mxproject`：STM32CubeMX 工程元数据
  - `Core/`：STM32CubeMX 自动生成的主程序入口、外设初始化代码和中断相关代码
  - `Drivers/`：STM32 HAL / CMSIS 等底层支持文件
  - `Middlewares/`：中间件代码，例如 FreeRTOS、USB 等
  - `USB_DEVICE/`：USB 设备相关自动生成代码
  - `MDK-ARM/RTE/`、`MDK-ARM/DebugConfig/`：Keil 相关自动生成配置

- **自定义代码部分**
  - `MDK-ARM/Drivers/`：放驱动层自定义代码
  - `MDK-ARM/App/`：放应用层自定义代码
  - `MDK-ARM/Protocol/`：放协议层自定义代码

简单理解就是：`Core/`、`Drivers/`、`Middlewares/`、`USB_DEVICE/` 这些更多是工程框架和自动生成内容；真正需要同学们重点阅读和编写的课程代码，主要放在 `App`、`Drivers`、`Protocol` 这些自定义目录中。

## 总览

本课程的主线：机电控制软件系统怎么拆解成启动项、驱动、感知、通信交互等模块；每个模块的作用是什么；最后怎么把这些模块组合成一个完整的 XY 工作平台控制系统。

### 课程主线 01 ~ 05

1. **实验 01：机电控制软件系统架构与开发环境**  
   从控制板认识和开发环境入手，先理解一个机电控制软件系统怎样拆解成启动项、驱动、感知、通信交互等模块。
2. **实验 02：步进电机直线模组驱动**
   聚焦驱动模块，理解执行器如何被控制。
3. **实验 03：接近开关感知**
   聚焦感知模块，理解传感器信息如何进入控制系统。
4. **实验 04：USB 通信与交互** 
   聚焦通信与交互模块，理解控制系统如何与上位机建立数据通信，以及上位机如何设计人机交互界面。
5. **实验 05：XY 工作平台插补运动控制系统**  
   将前面的驱动、感知、通信与交互模块组合起来，形成完整的 XY 工作平台控制系统。

### 课程外参考案例

`06+` 案例为课程外参考案例，供同学们在完成主线实验后继续扩展阅读和练习。

## 实验 01：机电控制软件系统架构与开发环境

**教学目标**  
认识教学板，搭建开发环境，并以 XY 工作平台为例理解一个机电控制软件系统怎样拆解成启动项、驱动、感知、通信交互几个模块。

**前置条件**  
教学板接好 LCD 屏幕；电脑已安装 `STM32CubeMX` 与 `Keil MDK`；已阅读 `Tutorial Board 原理图.pdf` 和 `Hardware/Beihang Mechatronics Tutorial Board Manual v1.1.pdf`。

**实验效果**  
程序烧录后，LCD 屏幕绘制北航校徽；同时完成开发环境的初次使用。

**当前工程入口**  
`Firmware_Example/01_Hello_Tutorial_Board/01_Hello_Tutorial_Board.ioc`

**操作内容**

1.   将 STM32CubeMX 的`.ioc`文件移动到你的空源码文件夹中
2.   使用 STM32CubeMX 配置好工程生成设置，并生成 Keil-MDK 工程
3.   在 Keil 工程中加入 `\01_Hello_Tutorial_Board\MDK-ARM\LowLayer` 文件夹中的 LCD 屏幕硬件驱动文件以及 `\01_Hello_Tutorial_Board\MDK-ARM\App` 文件夹中的北航校徽 RGB 值文件，并添加相应的头文件路径
4.   编译并烧录程序，看到实验结果

**掌握内容**  
1.   STM32 的开发环境（STM32CubeMX + Keil-MDK）
2.   STM32CubeMX 基础操作，包括：
	-   配置时钟树
	-   配置 GPIO
	-   配置外设
	-   生成代码和 Keil-MDK 工程
3.   Keil-MDK 基础操作，包括：
	-   导入程序文件
	-   配置头文件路径
	-   编译、烧录和调试程序
4.  机电控制软件系统的模块拆解思路
	-   启动项：系统启动时首先执行的代码，负责系统初始化和各模块的启动
	-   驱动模块：负责控制执行器（如电机）的代码
	-   感知模块：负责处理传感器输入的代码
	-   通信与交互模块：负责与上位机通信和人机交互的代码

## 实验 02：步进电机直线模组驱动

**教学目标**  
围绕驱动模块展开，通过步进电机与直线模组的控制案例，理解控制系统中执行层的编程方法和作用。

**实验效果**  
通过按键控制步进电机直线模组的梯形加减速运动。

**操作内容**
1.   将 STM32CubeMX 的`.ioc`文件移动到你的空源码文件夹中
2.   在 STM32CubeMX 中配置定时器、GPIO 等外设，配置 FreeRTOS 任务，并生成 Keil-MDK 工程
3.   在 Keil 工程中编写步进电机直线模组的驱动程序，主要包括：
	-   定时器中断服务和 PWM 输出调频函数，用于控制步进电机的速度
	-   GPIO 输出函数，用于控制步进电机的方向
	-   梯形加减速控制逻辑，用于实现平滑的运动控制
	-   参考 `\02_Stepper_Linear_Module_Drive\MDK-ARM` 中的实验程序文件
4.   调试程序，观察步进电机直线模组的驱动效果

**掌握内容**  
1. 使用 STM32 定时器和 GPIO 控制步进电机的基本方法  
2. 实现梯形加减速控制的基本思路
3. 主控制循环的构成：读取当前步进电机状态 -> 计算下一步的控制输出 -> 更新定时器和 GPIO 状态 -> 保存当前状态以供下一次循环使用
4. 驱动模块在整套机电控制系统中的位置

## 实验 03：接近开关感知

**教学目标**  
围绕感知模块展开，通过接近开关的输入处理案例，理解控制系统中感知层的编程方法和作用。

**实验效果**  
通过接近开关的输入控制步进电机直线模组的回零运动和限位保护。

**操作内容**
1.   将 STM32CubeMX 的`.ioc`文件移动到你的空源码文件夹中
2.   使用 STM32CubeMX 中配置 GPIO 外设，设置接近开关输入引脚为输入模式，配置 FreeRTOS 任务，并生成 Keil-MDK 工程
3.   在 Keil 工程中编写接近开关的感知程序，主要包括：
	-   接近开关状态轮询函数，用于定期更新接近开关的输入状态
	-   接近开关状态改变的事件处理逻辑，实现接近开关传感信息传递到主控制循环中
	-   在主控制循环中，根据接近开关的输入状态控制步进电机直线模组的回零运动和限位保护逻辑
	-   参考 `\03_Proximity_Switch_Sensing\MDK-ARM` 中的实验程序文件
4.   编译并烧录程序，观察接近开关输入与系统响应的实验现象

**掌握内容**  
1. 使用 STM32 GPIO 处理接近开关等离散感知信号的基本方法
2. 实现接近开关输入状态的轮询和事件处理的基本思路
3. 传感信息在主控制循环中的传递和使用
4. 感知模块与执行模块之间的配合关系

## 实验 04：USB 通信与交互

**教学目标**  
围绕 USB 通信与交互模块展开，理解控制板如何与上位机建立通信链路，以及上位机如何设计人机交互界面。

**实验效果**  
通过 USB 通信实现控制板与上位机的数据交互，并在上位机界面上控制步进电机直线模组的回零、加减速运动。

**操作内容**
1.   将 STM32CubeMX 的`.ioc`文件移动到你的空源码文件夹中
2.   使用 STM32CubeMX 配置 USB 外设，设置为 CDC（通信设备类）模式，配置 FreeRTOS 任务，并生成 Keil-MDK 工程
3.   在 Keil 工程中编写 USB 通信与交互程序，主要包括：
	-   USB CDC 初始化函数，用于设置 USB 通信接口
	-   数据协议处理函数，用于定义控制板与上位机之间的数据交换格式
	-   USB 数据接收和发送函数，用于实现控制板与上位机之间的数据交互
	-   在主控制循环中，根据从上位机接收到的命令控制步进电机直线模组的回零、加减速运动逻辑，并将当前状态通过 USB 发送回上位机
	-   上位机界面设计，使用 Python 或其他工具设计一个简单的 GUI 界面，通过 USB 发送控制命令给控制板，并显示当前状态
	-   参考 `\04_USB_Communication_And_Interaction\MDK-ARM` 和 `\04_USB_Communication_And_Interaction\gui_py` 中的实验程序文件
4.   编译并烧录程序，连接上位机，观察 USB 通信与交互的实验现象

**掌握内容**  
1. 使用 STM32 的 USB CDC 模式实现控制板与上位机的通信的基本方法
2. 设计控制板与上位机之间的数据协议的基本思路
3. 实现 USB 数据接收和发送的基本方法
4. 上位机 GUI 设计的基本思路 
5. 通信与交互模块在整套机电控制系统中的位置，以及它与驱动、感知模块的配合关系

## 实验 05：XY 工作平台插补运动控制系统

**教学目标**  
把前面的驱动、感知、USB 通信与交互模块组合起来，形成完整的 XY 工作平台插补运动控制系统。

**前置条件**  
已理解实验 01 中的系统分解思路；教学板与 XY 平台、限位和 USB 连接正确；已完成实验 02、03、04 中的模块编程练习。

**实验效果**  
通过上位机控制 XY 平台运动，完成回零、点位控制、直线插补和圆弧插补等功能，并在 GUI 上显示当前状态。

**当前工程入口**  
`Firmware_Example/05_XY_Platform_Motion_Control/XY_Platform_Motion_Control.ioc`

**操作内容**
1.   将 STM32CubeMX 的`.ioc`文件移动到你的空源码文件夹中
2.   将实验 02、03、04 中的模块配置（定时器、GPIO、USB、FreeRTOS 任务等）合并到这个新的 `.ioc` 文件中，并生成 Keil-MDK 工程
3.   将实验 02、03、04 中的模块程序（步进电机驱动、接近开关感知、USB 通信与交互）整合到这个新的工程中，并编写插补算法，形成一个完整的 XY 工作平台控制系统程序，主要包括：
	-   运动控制模块：读取当前状态 -> 处理 USB 命令 -> 处理感知输入 -> 更新驱动输出 -> 记录当前状态 -> 将状态传递给通信模块
	-   感知模块：读取接近开关状态 -> 将状态传递给运动控制模块
	-   通信模块：处理 USB 数据接收和发送 -> 解析上位机命令 -> 将命令传递给运动控制模块
	-   参考 `\05_XY_Platform_Motion_Control\MDK-ARM` 中的实验程序文件
4.   上位机界面设计，使用 Python 或其他工具设计一个 GUI 界面，通过 USB 与控制板建立通信，发送控制命令，并显示当前状态
	-   参考 `\05_XY_Platform_Motion_Control\gui_py` 中的实验程序文件
5.   编译并烧录程序，连接上位机，依次观察回零、点位控制、直线插补和圆弧插补的实验效果

**掌握内容**  
1. 控制、感知、通信与交互模块如何组合成完整系统  
2. XY 平台回零、点位控制与插补运动的基本控制逻辑  
3. 从单模块实验走向系统集成实验的整体思路

## 课程外参考案例

### 直流有刷电机 PID 控制（06+ 参考方向之一）

当前目录：`Firmware_Example/06_BDC_Motor_Control_PID/`

**前置条件**  
教学板已连接直流有刷电机、驱动与编码器；已具备 `STM32CubeMX` 与 `Keil` 的基本使用能力。

**实验效果**  
使用 STM32 读取 AB 相增量式编码器，通过 PID 控制器完成直流有刷电机的速度、位置闭环控制。

**当前工程入口**  
`Firmware_Example/06_BDC_Motor_Control_PID/06_BDC_Motor_Control_PID.ioc`

**操作内容**
1.   配置STM32定时器的编码器硬件解码功能，通过STM32获取编码器信息
2.   通过编码器数据计算电机的位置、速度。注意解决：定时器的计数器溢出、低速时的速度离散
3.   使用开环控制直流有刷电机（设置电压），观察其速度变化
4.   进行直流有刷电机的PID速度闭环控制
5.   进行直流有刷电机的位置、速度环级联控制

**掌握内容**  
1.   STM32 定时器的编码器模式使用
2.   闭环控制与开环控制的差别
3.   PID 闭环速度控制器结构
4.   位置、速度级联控制结构
5.   PID 参数对控制效果的影响以及调参方法
